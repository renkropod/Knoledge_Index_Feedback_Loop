# 4. 지식 노트

## 4-1. 오늘 새로 등장한 개념 3 개
1. **Agentic AI (에이전트 AI)**: 단순한 응답 생성을 넘어, 목표를 달성하기 위해 스스로 계획을 세우고 도구를 사용하며 상태를 유지하는 AI 시스템. (관련 글: Nvidia Vera CPU, Apideck CLI)
2. **Context Consumption (컨텍스트 소비)**: AI 에이전트가 외부 도구와 상호작용하거나 작업을 수행할 때 소모하는 입력/출력 토큰의 양. 효율성 저하의 주요 원인으로 지목됨. (관련 글: Apideck CLI)
3. **Distributed Systems for LLM Teams (LLM 팀을 위한 분산 시스템)**: LLM 개발 조직을 하나의 분산 시스템으로 간주하여, 팀 간 통신, 동기화, 자원 할당을 시스템 엔지니어링 관점에서 최적화하는 개념. (관련 글: Language model teams as distributed systems)

## 4-2. 꼭 알아야 할 배경 개념 3 개
1. **MCP (Model Context Protocol)**: AI 모델이 외부 데이터와 도구에 접근하기 위한 표준 프로토콜. 오늘 리포트에서 Apideck 가 이를 대체하거나 보완하려는 시도로 언급됨.
2. **Cursor AI**: AI 기반 코드 에디터로, 오늘 리포트에서 오픈소스 프로젝트의 품질 저하와 연관되어 논의됨.
3. **Godot**: 오픈소스 게임 엔진으로, Claude Code 를 통한 완전한 게임 빌드 사례에서 주요 플랫폼으로 등장함.

## 4-3. 도구/시스템 카드 3 개
1. **Apideck CLI**: MCP 대비 낮은 컨텍스트 소비를 목표로 하는 AI 에이전트 인터페이스 도구. (주요 용도: 에이전트 도구 연동, 대안: MCP, URL: hacker_news 내 링크)
2. **Nvidia Vera CPU**: 에이전트 AI 워크로드에 특화된 CPU. (주요 용도: 에이전트 추론 및 상태 관리, 대안: 기존 GPU 기반 추론, URL: Nvidia 공식 발표)
3. **Claude Code**: Anthropic 의 AI 코딩 도구. (주요 용도: 코드 생성, 게임 빌드, 대안: Cursor AI, URL: Show HN 링크)

## 4-4. 열린 연구 질문 3 개
